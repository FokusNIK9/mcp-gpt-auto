# Security: Secret Redaction

Для обеспечения безопасности ваших данных в проекте реализована система автоматического скрытия (redaction) секретов.

## Что скрывается автоматически

Система отслеживает и заменяет на `[REDACTED]` следующие данные в логах, отчётах и выводах команд:

- **GitHub Tokens**: классические (`ghp_...`) и fine-grained (`github_pat_...`).
- **OpenAI API Keys**: ключи вида `sk-...`.
- **GitLab Tokens**: `glpat-...`.
- **Присваивания в ENV**: любые строки вида `GITHUB_TOKEN=...`, `API_KEY=...`, `PASSWORD=...` и т.д.
- **Приватные ключи**: блоки `-----BEGIN PRIVATE KEY----- ... -----END PRIVATE KEY-----`.

## Как это работает

1. **Audit Logs**: Все данные, записываемые в `.agent/logs/audit.jsonl`, проходят через фильтр.
2. **Task Reports**: Markdown-отчёты в `.agent-queue/reports/` очищаются перед записью.
3. **Task Results**: JSON-файлы в `.agent-queue/done/` и `failed/` не содержат секретов.
4. **Console Output**: Вывод раннера в терминал также фильтруется.

## Правила безопасной работы

### 1. Не вставляйте токены в чат
Никогда не копируйте реальные значения `GITHUB_TOKEN` или других ключей в чат с ChatGPT или в файлы инструкций `prompt.md`.

### 2. Используйте переменные окружения
Запускайте раннер, устанавливая токен только в текущей сессии терминала:

**PowerShell:**
```powershell
$env:CONFIRM_PUSH="YES"
$env:GITHUB_TOKEN="[REDACTED]" # Здесь вставьте ваш реальный токен
scripts\win\21-run-task-runner-loop.bat
```

### 3. Если секрет был раскрыт
Если вы заметили, что токен попал в историю Git или в публичный лог:
1. **Revoke/Rotate**: Немедленно отзовите токен в настройках сервиса (GitHub, OpenAI и т.д.).
2. **Cleanup**: Удалите файл с секретом, если он не закоммичен.
3. **History**: Если секрет попал в коммит, рекомендуется отозвать токен, так как переписывание истории Git (`force push`) может нарушить работу других участников.

## Проверка системы redaction

Вы можете проверить работу фильтра командой:
```bash
npm run build
npm run test:redaction
```
